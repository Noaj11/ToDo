# 🚀 Laravel ToDo App

A modern, responsive ToDo application built with **Laravel 10** and **Bootstrap 5**.

## ✨ Features

- ✅ **Full CRUD Operations**
- ✅ **Toggle Complete/Incomplete**
- ✅ **Priority Levels** (Low/Medium/High)
- ✅ **Pagination**
- ✅ **Form Validation**
- ✅ **Responsive Design**
- ✅ **Flash Messages**
- ✅ **Confirmation Dialogs**

## 🛠️ Tech Stack









Git হচ্ছে বর্তমানে ডেভেলপারদের জন্য সবথেকে জনপ্রিয় ভার্সন কন্ট্রোল সিস্টেম। আপনি যে কমান্ডগুলোর কথা বলেছেন, সেগুলো একটি প্রজেক্টের শুরু থেকে শেষ পর্যন্ত ম্যানেজ করতে প্রতিনিয়ত কাজে লাগে। নিচে এগুলোর সহজ ব্যাখ্যা দেওয়া হলো:
### ১. সাধারণ টার্মিনাল কমান্ড (General Commands)
এই কমান্ডগুলো সরাসরি গিটের নয়, তবে কোড করার সময় টার্মিনালে এগুলো খুব বেশি প্রয়োজন হয়।
 * **mkdir (Make Directory):** নতুন কোনো ফোল্ডার বা ডিরেক্টরি তৈরি করার জন্য এটি ব্যবহার করা হয়। যেমন: mkdir my-project।
 * **touch:** নতুন কোনো ফাইল তৈরি করার জন্য এটি ব্যবহৃত হয়। যেমন: touch index.html।
 * **clear:** টার্মিনালের স্ক্রিন অনেক বেশি কমান্ডে ভরে গেলে তা পরিষ্কার করার জন্য এটি ব্যবহার করা হয়।
### ২. গিট সেটআপ এবং ট্র্যাকিং (Initialization & Tracking)
 * **git init:** কোনো প্রজেক্ট ফোল্ডারকে গিট রিপোজিটরি হিসেবে শুরু করার জন্য এই কমান্ড দিতে হয়। এটি ওই ফোল্ডারে একটি লুকানো .git ফোল্ডার তৈরি করে।
 * **git status:** আপনার প্রজেক্টের বর্তমান অবস্থা দেখার জন্য এটি ব্যবহৃত হয়। কোন ফাইলগুলো পরিবর্তন হয়েছে, কোনগুলো স্টেজ করা হয়নি—সবই এখানে দেখা যায়।
 * **git add .:** প্রজেক্টের সব পরিবর্তিত ফাইলকে **Staging Area**-তে পাঠানোর জন্য এটি ব্যবহৃত হয় (যাতে সেগুলোকে পরে সেভ বা কমিট করা যায়)।
 * **git add <filename>:** (আপনি git add() লিখেছেন, আসলে এটি নির্দিষ্ট ফাইলের নাম নেয়)। নির্দিষ্ট কোনো ফাইলকে স্টেজ করার জন্য এটি ব্যবহার করা হয়।
### ৩. পরিবর্তন সেভ করা (Saving Changes)
 * **git commit -m "message":** স্টেজ করা ফাইলগুলোকে স্থায়ীভাবে সেভ করার জন্য এটি ব্যবহার করা হয়। -m দিয়ে একটি মেসেজ দিতে হয় যাতে বোঝা যায় এই ধাপে কী কাজ করা হয়েছে।
 * **git commit -a:** এটি মূলত git add এবং git commit এর সমন্বয়। আগে থেকে ট্র্যাক করা ফাইলগুলোতে কোনো পরিবর্তন করলে এটি সরাসরি স্টেজ এবং কমিট করে ফেলে। তবে নতুন ফাইলের ক্ষেত্রে এটি কাজ করে না।
### ৪. ইতিহাস ও পার্থক্য দেখা (History & Comparison)
 * **git log:** প্রজেক্টে এ পর্যন্ত কতগুলো কমিট করা হয়েছে এবং কে করেছে, তার পুরো ইতিহাস দেখার জন্য এটি ব্যবহার করা হয়।
 * **git diff:** দুটি ফাইল বা কমিটের মধ্যে ঠিক কী কী পরিবর্তন (লাইন যোগ বা বিয়োগ) হয়েছে, তা বিস্তারিত দেখার জন্য এই কমান্ড ব্যবহার করা হয়।
### ৫. ব্রাঞ্চ এবং নেভিগেশন (Branching & Navigation)
 * **git branch:** প্রজেক্টের সব ব্রাঞ্চের লিস্ট দেখতে বা নতুন ব্রাঞ্চ তৈরি করতে এটি ব্যবহার করা হয়।
 * **git checkout:** এক ব্রাঞ্চ থেকে অন্য ব্রাঞ্চে যাওয়ার জন্য বা কোনো ফাইলের আগের ভার্সনে ফিরে যাওয়ার জন্য এটি ব্যবহৃত হয়। (বর্তমানে অনেক ক্ষেত্রে git switch ব্যবহার করা হয়)।
 * **git -f (Force Flag):** এটি মূলত একটি 'ফ্ল্যাগ'। কোনো কমান্ড জোরপূর্বক চালানোর জন্য এটি ব্যবহার করা হয় (যেমন: git push -f বা git branch -D)। এটি সাবধানে ব্যবহার করা উচিত কারণ এতে ডেটা হারানোর ভয় থাকে।
 * **git merge:** একটি ব্রাঞ্চের কোড অন্য আরেকটি ব্রাঞ্চের (যেমন: master বা main) সাথে যোগ করার জন্য এটি ব্যবহার করা হয়।
### ৬. রিমোট রিপোজিটরি (Remote Operations)
 * **git remote add origin <url>:** আপনার লোকাল রিপোজিটরিকে অনলাইন কোনো সার্ভারের (যেমন: GitHub) সাথে যুক্ত করার জন্য এটি ব্যবহার করা হয়।
 * **git push -u origin <branch-name>:** আপনার লোকাল কোড অনলাইনে পাঠানোর জন্য এটি ব্যবহার করা হয়। -u দিয়ে আপনার লোকাল ব্রাঞ্চকে রিমোট ব্রাঞ্চের সাথে ট্র্যাক করে রাখা হয়।
 * **git clone <url>:** অনলাইনে থাকা কোনো প্রজেক্ট হুবহু আপনার কম্পিউটারে কপি করে নিয়ে আসার জন্য এটি ব্যবহার করা হয়।
 * **git pull origin master:** রিমোট সার্ভার (GitHub) থেকে লেটেস্ট কোড এনে আপনার লোকাল প্রজেক্ট আপডেট করার জন্য এটি ব্যবহার করা হয়।
### ৭. বিশেষ কনসেপ্ট (Special Concepts)
 * **.gitignore:** এটি একটি ফাইল। আপনি যদি চান আপনার প্রজেক্টের কিছু ফাইল (যেমন: পাসওয়ার্ড ফাইল বা বড় লাইব্রেরি) গিটে সেভ না হোক, তবে সেই ফাইলের নামগুলো এখানে লিখে রাখতে হয়।
 * **flow (Git Flow):** এটি মূলত বড় প্রজেক্ট ম্যানেজ করার একটি নিয়ম বা স্ট্র্যাটেজি। এতে বিভিন্ন কাজের জন্য আলাদা আলাদা ব্রাঞ্চ (যেমন: develop, feature, release, hotfix) ব্যবহার করে কাজ গুছিয়ে রাখা হয়।
